帮助与支持

最后更新日期：2026年4月2日

Melodic Stamp 是一款专为 macOS 设计的原生音乐播放器，完全运行于沙盒环境，不扫描无关目录，不联网收集数据。它拥有强大的音频兼容性，并深度支持 TTML 逐字歌词、LRC 增强型翻译/注音，为您提供纯净、沉浸的本地音乐体验。

安装方式

  • App Store 正式版 — 稳定可靠，通过 macOS App Store 直接获取。
  • TestFlight 测试版 — 抢先体验新功能，名额有限，需接受潜在不稳定风险。加入测试前建议备份媒体库。
    注意：测试版与正式版可相互替换（构建号后两位高者可覆盖，否则需卸载重装）。

音频格式支持

Melodic Stamp 兼容几乎所有主流无损及有损格式，无需转码：

FLACMP3Ogg Opus/VorbisDSF / DSDIFF (DSD64, DoP) Monkey‘s Audio (APE)WavPackTrue AudioShorten WAV / AIFFMusepacklibsndfile 系列

歌词格式支持

精准显示原文、翻译、注音，支持逐字点亮与多轨对唱。

  • TTML — 逐字 / 逐行歌词，支持对唱、背景歌词，兼容 AMLL TTML Database 与 Apple Music 逐字注音规范。
  • LRC — 基础歌词、逐字型 LRC、增强型 LRC（社区扩展格式：支持原文 + 翻译 + 注音连续或分段排列）。
    注：LRC 若仅提供注音行而无翻译，注音会被识别为翻译，属格式限制。
  • 内嵌歌词 — 读取音频元数据（ID3 Lyrics）中的歌词。

快速上手

1. 导入音乐文件夹
执行以下任一操作：
• 快捷键 Command + L 打开文件夹选择器，选中音乐目录。
• 点按“点击添加文件夹”按钮，或直接拖拽文件夹至导入区域。
Melodic Stamp 仅扫描您主动添加的文件夹，不会读取任何其他位置。

2. 歌词自动匹配
应用会在导入的文件夹（含子文件夹）内，查找与音频文件同名的歌词文件。
匹配优先级：.ttml 优先，其次为 .lrc 或 .txt 。若同时存在外置歌词与内嵌歌词，默认优先使用外置歌词（可在元数据编辑器中调整）。

3. 开始播放
点击曲目即可播放，媒体库支持收藏、歌单、专辑及艺术家视图。还可通过菜单栏图标快速控制或显示当前歌词。

隐私与安全核心

本应用 不请求任何网络权限，绝不收集个人信息或上传音频。所有数据仅存于本地沙盒，卸载即清除。元数据编辑器会直接修改音频文件，建议编辑前备份重要文件。

暂不支持 NAS、网络驱动器或挂载存储设备（稳定性考虑），计划后续通过 Jellyfin 等扩展网络库支持。

个性化设置

通过 设置 (⌘ ,) 可深度定制体验：

  • 窗口背景 — 纯色 / 活力 / 空灵，调整模糊与混色效果。
  • 歌词显示 — 独立开关翻译、注音，支持粗体、淡出效果，实验性歌词渲染器（TextRender）可提升逐字精度。
  • 菜单栏歌词 — 显示当前歌词或光碟图标，支持逐字动画（占用稍高，属正常现象）。
  • 媒体库 — 文件夹自动观察、启动时刷新，可强制重新扫描元数据。
  • 待播清单记忆 — 重启后恢复播放队列及进度。
  • 悬浮窗/常亮 — 专注歌词页时可防止屏幕休眠。

常见问题

  • 为什么无法导入 NAS 或挂载磁盘中的文件夹？
    为保证稳定，当前版本仅支持内置或直连本地存储。挂载设备存在读取延迟可能引发崩溃，后续将通过 Jellyfin 等方案支持网络库。
  • 开启菜单栏歌词后不显示或只显示图标？
    macOS 在菜单栏空间不足时会自动隐藏第三方图标。可减少菜单栏项，或关闭逐字歌词模式尝试。
  • LRC 歌词中的注音被错误归为翻译行怎么办？
    LRC 本身无标准注音字段，当缺少翻译行时系统将注音行映射为翻译。建议使用 TTML 歌词获得完美逐字注音体验，或确保 LRC 包含翻译行结构。
  • 歌词过滤开启后仍有部分创作者信息残留？
    过滤功能基于常见模式匹配，生僻或特殊字段可能遗漏，欢迎反馈歌词文件协助改进。
  • 是否支持均衡器？
    内置均衡器，可自定义频段，提升听感。

本应用不提供云端歌词下载或识别，所有歌词需自行准备。但完善的本地匹配机制让您的歌词库井然有序。如需反馈问题或功能建议，欢迎邮件联系。
